# Customer Messaging Automation (n8n)

This workflow fetches customer data from a datastore, processes the data, and sends automated messages to customers.

## Workflow Overview
The workflow is manually triggered to retrieve all customer records from the n8n datastore. The data is then edited or filtered before sending automated messages to customers.

## Tools Used
- n8n
- Datastore
- Manual Trigger
- Customer Messenger

## How It Works
1. Workflow starts manually using the Execute button  
2. Customer data is fetched from the datastore  
3. Data is processed using the Edit Fields node  
4. Automated messages are sent to customers  

## Use Case
- Customer follow-ups  
- Automated notifications  
- Customer engagement campaigns  
- Training and testing automation workflows  

## Result
- Automated customer communication  
- No manual messaging required  
- Faster response and engagement  
- Clean and structured customer data usage
